## 1. ВСТУП (INTRODUCTION)
RenderPen — це вебсервіс для створення анотацій та інтерактивних посібників на основі готових 3D-моделей. Продукт вирішує проблему компаній-виробників (OEM), яким потрібен простий онлайн-інструмент для створення багатокрокових інструкцій, підсвічування деталей та візуальних зрізів без використання складного CAD-програмного забезпечення. Цільова аудиторія: технічні письменники, інженери, розробники 3D-контенту.

---

## 2. ЗАГАЛЬНИЙ ОПИС (OVERALL DESCRIPTION)
**Актори:**
* Глядач- перегляд готових інтерактивних інструкцій, обертання та масштабування 3D-моделей за публічним посиланням.
* Зареєстрований користувач - завантаження GLB-файлів, створення анотацій, міток (labels), налаштування зрізів (cutaways) та публікація.
* Администратор - управління тарифами, обмеженнями сховища та обліковими записами.

**Залежності:**
Підтримка WebGL (рушій PlayCanvas або Three.js) для рендерингу 3D у браузері; хмарне сховище (наприклад, AWS S3) для збереження користувацьких 3D-моделей.

## 3. ФУНКЦІОНАЛЬНІ ВИМОГИ (FUNCTIONAL REQUIREMENTS)
* **FR-01 [Must]:** Система повинна дозволяти користувачу завантажувати 3D-моделі у форматі .glb.
* **FR-02 [Must]:** Система повинна дозволяти додавати текстові мітки (labels), прив'язані до конкретних координат на поверхні 3D-моделі.
* **FR-03 [Must]:** Користувач може створювати багатокрокові інтерактивні інструкції (сцени).
* **FR-04 [Should]:** Система підтримує функцію створення "зрізів" (cutaways) для демонстрації внутрішніх частин механізмів.
* **FR-05 [Should]:** Система дозволяє виділяти (підсвічувати) окремі компоненти моделі різними кольорами.
* **FR-06 [Could]:** Система дозволяє імпортувати технічні дані моделі з метаданих файлу GLB.
* **FR-07 [Must]:** Система повинна генерувати публічне вебпосилання для поширення готового посібника.
* **FR-08 [Must]:** Переглядач (Viewer) повинен підтримувати обертання, панорамування та масштабування моделі за допомогою миші або сенсорного екрана.

---

## 4. НЕФУНКЦІОНАЛЬНІ ВИМОГИ (NON-FUNCTIONAL REQUIREMENTS)
* **NFR-P-01 (Продуктивність):** Завантаження та рендеринг 3D-моделі розміром до 50 МБ має відбуватися не довше 5 секунд (при швидкості інтернету 50 Мбіт/с).
* **NFR-P-02 (Продуктивність):** Взаємодія з моделлю у браузері має відбуватися з частотою кадрів не менше 30 FPS.
* **NFR-S-01 (Безпека):** Усі збережені 3D-моделі є приватними за замовчуванням, доступ можливий лише за згенерованим токеном/посиланням.
* **NFR-U-01 (Зручність):** Режим перегляду інструкцій повинен бути повністю адаптивним і працювати на смартфонах без втрати функціоналу масштабування.
* **NFR-R-01 (Надійність):** Доступність опублікованих посібників (uptime) повинна складати не менше 99.9%.

---

## 5. ОБМЕЖЕННЯ (CONSTRAINTS)
* **C-01:** Рендеринг має відбуватися виключно стандартними засобами браузера (WebGL) без необхідності встановлення сторонніх плагінів.
* **C-02:** Основний формат файлів жорстко обмежений розширенням .glb (glTF). Інші CAD-формати вимагають попередньої конвертації користувачем.
* **C-03:** Обмеження на maximalний розмір завантажуваного файлу залежить від тарифного плану користувача.

---

## 6. ГЛОСАРІЙ (GLOSSARY)
* **GLB (glTF)** — стандартний бінарний формат файлу для 3D-сцен і моделей, що містить геометрію, текстури та матеріали.
* **OEM (Original Equipment Manufacturer)** — компанія, яка розробляє та виготовляє продукцію або комплектуючі.
* **Cutaway (Зріз)** — візуальний ефект в інструменті, який дозволяє "зрізати" зовнішні шари моделі для показу її внутрішньої структури.
* **Label (Мітка/Етикетка)** — текстова або візуальна підказка, зафіксована на певній деталі 3D-моделі.
* **WebGL** — технологія для відтворення інтерактивної 3D-графіки в межах будь-якого сумісного веббраузера.
